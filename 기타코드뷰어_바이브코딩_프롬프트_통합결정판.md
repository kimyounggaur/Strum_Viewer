## 0. 영상에서 확인한 핵심 UI / 동작

첨부 영상을 다음과 같이 해석해 구현합니다.

1. 전체 배경은 흰색(또는 아주 밝은 회색)이며, 부드러운 그림자의 **뉴모피즘** 톤이다.
2. 상단: 왼쪽에 **돋보기 아이콘 + 길고 둥근(pill) 검색창**, 오른쪽에 연한 핑크색 **`Lesson Designer`** 타이틀(글자 간격 넓음).
3. 첫 화면에 코드 종류를 고르는 **원형 버튼 12개**가 2줄로 배치된다.
   - 1행: `Major` · `7` · `minor` · `minor7` · `sus4` · `Major 7`
   - 2행: `6` · `7sus4` · `add2` · `m7(b5)` · `diminish` · `Augment`
4. 각 원형 버튼은 **흰색 내부 + 색상 외곽선(링) + 부드러운 그림자**로 살짝 떠 있는 느낌이다.
5. 버튼 위에 **클릭 포인터(손가락) 커서**가 보이며, hover/클릭 시 **부드럽게 살짝 커진다**.
6. `Major`를 클릭하면 Major 코드 카드 목록으로 전환된다.
7. 카드 목록에는 `C Major`, `D Major`, `E Major`, (`F Major`,) `G Major`, `A Major`, (`B Major`) 등이 보인다. *(영상은 C·D·E·G·A를 부각)*
8. 각 코드 카드는 **작은 기타 코드 다이어그램**과 라벨을 가진다.
9. 특정 코드(예: `C Major`)를 클릭하면 화면 중앙에 코드명이 **크게** 뜨고, 아래에 **큰 코드 다이어그램**이 나타난다. 선택되지 않은 코드들은 보조 목록/썸네일로 남는다.
10. 모든 전환(버튼 확대, 카드 등장, 상세 줌인)은 **부드러운 transition/animation**이 있다.

**3단계 화면 흐름 (Gemini식 ‘수준’ 명명)**
```
[수준1: 메인 - 코드 종류 선택] ──클릭──▶ [수준2: 갤러리 - 루트음 선택] ──클릭──▶ [수준3: 상세 - 대형 다이어그램]
        ▲                                          │                                       │
        └──────────────── 뒤로 / 홈(로고) 버튼 ◀────┴───────────────────────────────────────┘
```

## 1. 마스터 프롬프트 (한 번에 시작용)

> 아래 블록을 그대로 복사해 AI 코딩 도구의 **첫 메시지**로 붙여넣으세요.

```text
너는 React, TypeScript, Vite, TailwindCSS에 능숙한 시니어 프론트엔드 개발자다.
첨부 영상과 같은 "기타 코드 뷰어 웹사이트(Lesson Designer)"를 만들어라.

[핵심 컨셉]
- 기타 학습자가 코드 운지법을 보기 좋게 모아 보는 레퍼런스 SPA.
- 3단계 흐름:
  ① 메인: 코드 "종류"를 고른다 (Major, 7, minor, minor7, sus4, Major 7, 6, 7sus4, add2, m7(b5), diminish, Augment).
  ② 갤러리: 고른 종류의 코드들(C, D, E, F, G, A, B…)이 카드 그리드로 펼쳐진다.
  ③ 상세: 특정 코드를 누르면 크게 확대된 운지 다이어그램을 본다.
- 코드 운지는 "가로로 누운 프렛보드 다이어그램"으로 보여준다.

[디자인 톤]
- 흰색 배경 + 부드러운 그림자(뉴모피즘).
- 코드 종류 버튼은 색깔 있는 동그란 링(circle) 버튼, 알록달록.
- 상단에 돋보기 검색창 + "Lesson Designer" 로고(연핑크).

[기술 스택 — 고정]
- React + TypeScript + Vite + TailwindCSS.
- 화면 전환은 별도 라우터 없이 React state로 관리해도 된다.
- 백엔드 없이 정적 사이트로 동작해야 한다 (Vercel/Netlify/GitHub Pages 배포 가능).

[⭐ 코드 이미지 처리 — 가장 중요]
- 코드 다이어그램을 사이트 안에서 관리하는 구조를 반드시 포함하라.
- 기본 전략은 "데이터(운지 정보) → 동적 SVG 렌더링"으로 한다. (이미지 파일 관리·저작권 부담을 없앰)
- 동시에 정적 이미지 폴백도 지원한다: chord.image 경로가 있으면 그 이미지를, 없으면 SVG를 그린다.
- 사용자가 직접 이미지를 추가하는 기능은 IndexedDB(브라우저 저장)로, 운영 확장은 Supabase Storage로 분기할 수 있게 저장 로직을 lib/에 격리하라.
- "정적 배포된 웹사이트의 public 폴더는 브라우저에서 직접 수정할 수 없다"는 점을 코드 주석과 README에 명시하라.

[진행 방식]
먼저 ① 폴더 구조와 ② 전체 컴포넌트 설계를 제안하라.
내가 확인하면, 타입/데이터 → 동적 SVG Fretboard → 헤더 → 코드 버튼 → 갤러리 → 상세 → 상태연결 → 검색 → 업로드 → 마무리 순으로 단계별 구현하라.
TypeScript any는 쓰지 말고, 컴포넌트를 역할별로 분리하라.
```

## 4. 화면 구성 & 사용자 흐름 (3단계)

### 4.1 수준1 — 메인 (코드 종류 선택)
- 상단 헤더: 돋보기 아이콘 + 둥근 검색창(연핑크 테두리) + 우측 `Lesson Designer`.
- 본문: **코드 종류 12개**를 **2행 × 6열** 원형 링 버튼으로 배치(§5.3 색상표).
- hover/클릭 시 부드럽게 확대(`scale`), active 상태는 더 크게 + 링 강조.

### 4.2 수준2 — 갤러리 (루트음 선택)
- 선택한 종류(예: Major)의 코드를 **카드 그리드**로 표시 (`C Major`, `D Major`, …).
- 상단에 현재 종류 타이틀 표시.
- 각 카드 = 코드명 라벨(초록 점선 박스 느낌) + 미니 프렛보드.
- **첫 번째(또는 선택된) 코드 1개는 크게 강조**, 나머지는 작게.
- 카드 클릭 → 수준3로 줌인 전환.

### 4.3 수준3 — 상세 (대형 다이어그램)
- 상단 중앙에 **큰 코드명 제목 박스**(초록 텍스트 + 주황 테두리 느낌).
- 그 아래 **크게 확대된 프렛보드** 하나.
- 좌상단 루트 라벨(예: `|C`).
- 우측(데스크톱) 또는 하단 가로 스크롤(모바일)에 같은 종류의 다른 코드 썸네일 → 클릭 시 즉시 전환.
- **뒤로/홈 버튼**, **ESC 키**로 갤러리 복귀.

## 5. 디자인 명세

### 5.1 전체 톤
- 배경: `#FFFFFF`~`#FBFBFB`.
- **뉴모피즘**: 위쪽 하이라이트 + 아래쪽 부드러운 그림자.
  - 예: `box-shadow: 6px 6px 14px rgba(0,0,0,.08), -6px -6px 14px rgba(255,255,255,.9);`
- 넓은 여백, 밝고 컴팩트한 레이아웃.

### 5.2 헤더
- 왼쪽: 돋보기 아이콘(연핑크) + **pill 검색창**(연핑크 테두리, focus ring도 연핑크).
- 오른쪽: **`Lesson Designer`** — 연핑크, letter-spacing 넓게.
- 모바일: 브랜드 텍스트가 작아지거나 검색창 아래로 내려가도 됨.

### 5.3 코드 종류 버튼 (색상표)
각 버튼 = **흰 배경 + 색상 링(테두리) + 가운데 색 글자 + 뉴모피즘 그림자**.
아래 hex와 Tailwind 클래스는 영상 톤에 맞춰 조정한 값입니다.

| # | 라벨 | 색 톤 | hex | Tailwind(border/text) |
|---|------|------|-----|------------------------|
| 1 | `Major` | 주황 | `#f97316` | `border-orange-500` / `text-rose-800` |
| 2 | `7` | 노랑 | `#facc15` | `border-yellow-300` / `text-yellow-600` |
| 3 | `minor` | 연두 | `#bef264` | `border-lime-300` / `text-lime-600` |
| 4 | `minor7` | 초록 | `#10b981` | `border-emerald-400` / `text-emerald-600` |
| 5 | `sus4` | 하늘(시안) | `#22d3ee` | `border-cyan-400` / `text-cyan-600` |
| 6 | `Major 7` | 파랑 | `#0ea5e9` | `border-sky-500` / `text-sky-700` |
| 7 | `6` | 남색 | `#1e3a8a` | `border-blue-900` / `text-blue-900` |
| 8 | `7sus4` | 보라 | `#8b5cf6` | `border-violet-400` / `text-violet-700` |
| 9 | `add2` | 연분홍 | `#fda4af` | `border-rose-200` / `text-rose-400` |
| 10 | `m7(b5)` | 진분홍(푸시아) | `#f0abfc` | `border-fuchsia-300` / `text-fuchsia-600` |
| 11 | `diminish` | 보라 | `#a855f7` | `border-purple-600` / `text-purple-800` |
| 12 | `Augment` | 갈색/황금 | `#a16207` | `border-yellow-700` / `text-rose-800` |

> 라벨은 영상에 `diminsh`, `Augment`로 표기돼 있습니다. 가독성을 위해 `diminish`로 적었으니, 영상 그대로 원하면 `diminsh`로 바꾸세요. (정식 명칭은 `diminished`/`augmented`.)

### 5.4 코드 다이어그램(프렛보드) — 정밀 스펙 ⭐
**가장 핵심적인 시각 요소.** 영상은 "가로로 누운" 프렛보드입니다. SVG로 그릴 때 아래를 정확히 지키세요.

- **방향**: 줄(string)이 **수평**. **`1번줄`(가장 가는 줄)이 맨 위**, `6번줄`(가장 굵은 줄)이 맨 아래.
- **줄(string)**: 6개의 **빨강/크림슨** 가로선. 오른쪽 끝에 `1번줄`~`6번줄` 라벨(빨강).
- **프렛(fret)**: **초록** 세로선. 하단에 `1프렛` · `2프렛` · `3프렛` **초록 알약(pill) 라벨**.
- **너트(nut)**: 맨 왼쪽에 **두꺼운 회색 세로 막대**.
- **루트 라벨**: 좌상단에 코드 루트(예: `|C`), 파란색.
- **운지 표시**: 누르는 칸에 **흰색 둥근 사각형(손가락 캡슐)** + 그 위에 **검은 점**, 캡슐 안에 finger 번호(작게). 캡슐에 옅은 사선 빗금 텍스처(선택).
- **개방현/뮤트**: 줄 왼쪽 끝에 개방현 `o`, 안 치는 줄 `x` 표시.
- **제목 박스**(상세): 코드명(예 "C Major")을 **색 텍스트 + 같은 색 테두리 박스**로.

**SVG 좌표 가이드(예시, viewBox `0 0 560 320`)**
```
- 너트 x ≈ 70, 프렛 간격 ≈ 130px → 1프렛선 x≈200, 2프렛 x≈330, 3프렛 x≈460
- 줄 y: 1번줄(위) y≈60 … 6번줄(아래) y≈260, 간격 40px
- 운지 캡슐: 해당 줄 y, (fret n-1 ~ n) 중앙 x 에 배치 / 검은 점 r≈10
- 줄 라벨: x≈500 우측 / 프렛 라벨: 각 칸 중앙, y≈300
```

### 5.5 애니메이션
1. 원형 버튼 hover: `scale 1 → 1.06`, 그림자 강화
2. 원형 버튼 active: `scale 1.1`, 링 두께/글로우 강화
3. 종류 선택 → 갤러리 등장: `opacity 0→1`, `translateY 12px→0`
4. 코드 카드 hover: `scale 1 → 1.04`
5. 상세 진입: 큰 이미지 `opacity 0→1`, `scale .96→1`
6. 우측 썸네일 목록: staggered 등장 느낌
- 전부 `transition: 0.2~0.3s ease`. 필요 시 CSS keyframes 추가.

### 6.1 타입 정의 (`src/data/chordTypes.ts`)

```ts
  | 'major' | 'dominant7' | 'minor' | 'minor7' | 'sus4' | 'major7'
  | 'six' | 'sevenSus4' | 'add2' | 'm7b5' | 'diminish' | 'augment';

// fret:   0 = 개방현, N = N번 프렛
export type FingerPosition = {
  string: 1 | 2 | 3 | 4 | 5 | 6;
  fret: number;
  finger?: 1 | 2 | 3 | 4;            // 검지1 ~ 새끼4
  open?: boolean;                    // 개방현(o)
  muted?: boolean;                   // 안 치는 줄(x)
};

export type ChordShape = {
  id: string;                        // 'c_major'
  root: string;                      // 'C'
  quality: ChordQualityId;
  title: string;                     // 'C Major'
  positions: FingerPosition[];       // 6개 줄에 대한 운지

  difficulty?: 'beginner' | 'intermediate' | 'advanced';
  tags?: string[];
};
```

## 7. ⭐ 기타 코드 이미지 저장 방법 (핵심)

> 사용자가 가장 중요하게 요청한 부분. **기본 전략 = 7.1(동적 SVG) + 7.2(정적 폴백) 하이브리드**.
> 이미지 파일을 일일이 관리하기 싫으면 7.1만으로도 완성됩니다.

### 7.0 방법 비교표 & 선택 가이드

| 방법 | 이미지를 어디에? | 백엔드 | 추가 방법 | 적합한 경우 |
|------|------------------|--------|-----------|-------------|
| **A. 동적 SVG 생성** ⭐1순위 | 저장 안 함(데이터로 그림) | ❌ | `chords.ts`에 운지 데이터만 추가 | 관리 0·무한 확대·저작권 무관 |
| **B. 정적 파일 + manifest** ⭐기본 폴백 | `public/chords/...` | ❌ | 파일 넣고 manifest 수정 후 재배포 | 디자인된 이미지를 그대로 쓰고 싶을 때 |
| **C. IndexedDB 업로드** | 사용자 브라우저 내부 | ❌ | 사이트 업로드 UI | 재배포 없이 내 브라우저에서 추가/보관 |
| **D. 클라우드(Supabase 등)** | 클라우드 버킷 | ✅ | 관리자 업로드 → 전체 반영 | 여러 사용자 공유·관리자 기능 |

> 🎯 **고르기**: 정해진 코드 직접 관리 → **A(+B)**. 디자인 이미지 사용 → **B**. 재배포 없이 내가 추가 → **C**. 다 같이 공유 → **D**.

### 7.1 [1순위·권장] 데이터 기반 동적 SVG
운지 데이터(`positions`)를 받아 `Fretboard` 컴포넌트가 SVG를 직접 그립니다. **이미지 파일이 아예 필요 없습니다.**

`Fretboard.tsx` 스켈레톤:
```tsx
type Props = { shape: ChordShape; size?: 'thumb' | 'large' };

export function Fretboard({ shape, size = 'thumb' }: Props) {
  // 1) 이미지 폴백: shape.image 가 있으면 <img>로 그리고 종료 (§7.6 우선순위)
  // 2) 없으면 SVG 렌더:
  //    - 6개 가로선(빨강): string 1=위 ~ 6=아래
  //    - 세로 프렛선(초록) + 하단 "1프렛/2프렛/3프렛" 알약 라벨
  //    - 왼쪽 회색 너트
  //    - positions 순회: fret>0 → 해당 줄·칸 중앙에 흰 캡슐 + 검은 점 + finger 번호
  //                      open → 줄 왼쪽 'o', muted → 'x'
  //    - 좌상단 루트 라벨 "|{root}"
  //    - 색/좌표는 §5.4 가이드 따름
  return (/* <svg viewBox="0 0 560 320"> ... </svg> 또는 <img/> */);
}
```
**장점**: 파일 관리 0, 용량 극소, 색·크기 자유, 저작권 안전(직접 그림), 코드 수백 개도 데이터만 추가.
**단점**: 컴포넌트 초기 구현에 시간 듦(한 번 만들면 끝).

### 7.2 [기본 폴백] 정적 파일 + manifest.json
디자인된 다이어그램 이미지를 그대로 쓰고 싶을 때.

- **경로 규칙(고정)**: `public/chords/{quality}/{root}.svg`
  ```txt
  C Major   → /chords/major/c.svg
  C# Major  → /chords/major/c-sharp.svg   (# 은 -sharp 로 치환)
  Db Major  → /chords/major/d-flat.svg
  A minor7  → /chords/minor7/a.svg
  B m7(b5)  → /chords/m7b5/b.svg
  ```
- **manifest.json** (이미지 수십 개 이상일 때):
  ```json
  {
    "version": 1,
    "basePath": "/chords",
    "items": [
      { "id": "c_major", "root": "C", "quality": "major", "title": "C Major", "imageSrc": "/chords/major/c.svg" },
      { "id": "d_major", "root": "D", "quality": "major", "title": "D Major", "imageSrc": "/chords/major/d.svg" }
    ]
  }
  ```
- **표시**:
  ```tsx
  <img src={shape.image} alt={`${shape.title} 기타 코드 다이어그램`} loading="lazy"
       onError={(e) => { e.currentTarget.src = '/chords/placeholders/chord-placeholder.svg'; }} />
  ```
> 📌 포맷은 **SVG**(무한 확대) 또는 **WebP**(용량↓) 권장. 크기 통일(예 520×320).

### 7.3 [선택] IndexedDB 업로드 — 재배포 없이 브라우저 저장
이미지는 바이너리(Blob)라 `localStorage`보다 **IndexedDB**가 적합. `idb` 패키지 사용.

```bash
npm install idb
```
`src/lib/chordImageStorage.ts`:
```ts
import { openDB } from 'idb';

export type UploadedChordImage = {
  id: string; chordId: string; root: string; quality: string; title: string;
  fileName: string; mimeType: string; blob: Blob; createdAt: string; updatedAt: string;
};

const DB_NAME = 'guitar-chord-viewer';
const STORE = 'chord-images';

async function db() {
  return openDB(DB_NAME, 1, {
    upgrade(d) { if (!d.objectStoreNames.contains(STORE)) d.createObjectStore(STORE, { keyPath: 'id' }); },
  });
}
export async function saveUploadedChordImage(item: UploadedChordImage) { (await db()).put(STORE, item); }
export async function getUploadedChordImageUrl(id: string) {
  const item = await (await db()).get(STORE, id) as UploadedChordImage | undefined;
  return item ? URL.createObjectURL(item.blob) : null;   // <img src>에 사용
}
```
업로드 UI 규칙: 허용 형식 `svg/png/webp`, **2MB 이하**, 미리보기 표시, 저장 후 목록 즉시 반영, 새로고침 후 재로드, 삭제 지원.
**장점**: 백엔드 0, 새로고침해도 유지. **단점**: 그 브라우저/기기에서만 보임(공유 X).

### 7.4 [선택] Supabase Storage — 클라우드 공유·관리자 업로드
```ts
import { createClient } from '@supabase/supabase-js';
const supabase = createClient(URL, ANON_KEY);

await supabase.storage.from('chord-images').upload('major/c.svg', file, { upsert: true });
const { data } = supabase.storage.from('chord-images').getPublicUrl('major/c.svg');
const imageUrl = data.publicUrl;   // shape.image 에 그대로 연결
```
DB 테이블:
```sql
create table chord_images (
  id uuid primary key default gen_random_uuid(),
  chord_id text not null, root text not null, quality text not null, title text not null,
  storage_path text not null, public_url text not null,
  created_at timestamptz default now(), updated_at timestamptz default now()
);
```
앱 로딩 흐름: ① 정적 기본 데이터 로드 → ② Supabase에서 추가 메타데이터 fetch → ③ 같은 `chordId`면 클라우드 이미지가 정적 이미지를 **override**, 없으면 새 코드로 추가.

### 7.5 ⚠️ 반드시 알아야 할 현실적 한계
브라우저에서 이미지를 업로드해도 **배포된 정적 사이트의 `public/chords` 폴더에 파일이 써지지 않습니다.** 정적 사이트는 배포 후 서버 파일시스템을 브라우저가 직접 수정할 수 없습니다. 그래서 저장은 목적별로 나눕니다.
1. **개발자 기본 이미지**: `public/chords`에 직접 넣고 Git 관리.
2. **사용자 개인 추가**: IndexedDB(그 브라우저에서만).
3. **전체 공유**: Supabase/S3/R2 등 object storage + DB.

> README와 코드 주석에 다음 문장을 **반드시** 포함하세요:
> *"정적 배포된 웹사이트의 public 폴더는 브라우저에서 직접 수정할 수 없다. 기본 이미지는 개발자가 프로젝트에 포함해 배포하고, 사용자 업로드 이미지는 IndexedDB 또는 서버 스토리지에 저장해야 한다."*

### 7.6 하이브리드 표시 우선순위 (AI에게 고정)
`Fretboard`는 다음 순서로 표시합니다:
**① IndexedDB 업로드 이미지 → ② `shape.image`(정적/클라우드) → ③ 동적 SVG 렌더**.
저장 로직은 전부 `src/lib/chordImageStorage.ts`에 격리해, 나중에 IndexedDB ↔ Supabase 전환이 한 파일 수정으로 끝나게 합니다.

## 8. 상태 관리 설계

전역 상태 라이브러리 없이 React state + custom hook으로 충분합니다.
```ts
type AppView = 'qualities' | 'grid' | 'detail';

const [searchTerm, setSearchTerm] = useState('');
const [selectedQuality, setSelectedQuality] = useState<ChordQualityId | null>(null);
const [selectedChordId, setSelectedChordId] = useState<string | null>(null);
const [uploadedChords, setUploadedChords] = useState<ChordShape[]>([]);

const allChords = [...staticChords, ...uploadedChords];
const filtered = allChords.filter((c) => {
  const okQuality = selectedQuality ? c.quality === selectedQuality : true;
  const q = searchTerm.trim().toLowerCase();
  const okSearch = !q || c.title.toLowerCase().includes(q) || c.root.toLowerCase().includes(q)
    || c.quality.toLowerCase().includes(q) || c.tags?.some((t) => t.toLowerCase().includes(q));
  return okQuality && okSearch;
});
```
**전환 규칙**: 진입=종류만 강조 → 종류 클릭=`selectedQuality` 설정·grid 표시 → 카드 클릭=`selectedChordId` 설정·detail 표시 → 뒤로=detail→grid, grid→qualities. 검색어가 있으면 종류 미선택이어도 전체에서 검색.

## 10. 단계별 개발 프롬프트 (Step 1~12)

> 각 블록을 **하나씩** 순서대로 붙여넣으세요.

### Step 1 — 프로젝트 스캐폴딩
```text
React + TypeScript + Vite + TailwindCSS 프로젝트(guitar-chord-viewer)를 생성하라.
기본 예제 코드를 제거하고 §3 폴더 구조(src/components, data, hooks, lib, styles / public/chords)를 만들어라.
Tailwind를 App.tsx에서 정상 동작하게 설정하고 기본 배경을 #fbfbfb로, 한글 폰트(Pretendard)와 영문 폰트(Poppins)를 로드하라.
README에 실행 방법을 적어라.
```

### Step 2 — 타입 & 데이터
```text
src/data/chordTypes.ts, chordQualities.ts, chords.ts를 만들어라.
- §6.1 타입(ChordQualityId, FingerPosition, ChordShape)을 정의한다. any 금지.
- §6.2 chordQualities 12종(색상/Tailwind/설명 포함)을 영상 순서대로 정의한다.
- §6.3 형식으로 Major 코드 C·D·E·F·G·A·B를 ChordShape 배열로 정의한다.
  운지는 [부록 A] 표를 기준으로 정확히 채운다. (string 1=가는 줄, 6=굵은 줄 / 0=open / muted=x)
- 다른 종류는 아직 비어 있어도 된다.
```

### Step 3 — 동적 SVG Fretboard (⭐ 핵심)
```text
src/components/Fretboard.tsx 를 만들어라. props는 { shape: ChordShape, size?: 'thumb'|'large' }.
표시 우선순위: ① shape.image 있으면 <img> ② 없으면 SVG 렌더.
SVG 규칙(§5.4):
- 가로로 누운 프렛보드, viewBox "0 0 560 320".
- 6개 가로 줄(빨강): string 1=맨 위 ~ 6=맨 아래, 오른쪽에 '1번줄'~'6번줄' 라벨.
- 세로 프렛선(초록) + 하단 '1프렛/2프렛/3프렛' 초록 알약 라벨.
- 왼쪽 회색 너트(두꺼운 세로 막대).
- positions 순회: fret>0 → 해당 줄·칸 중앙에 흰 둥근 캡슐 + 검은 점 + finger 번호(작게);
  open → 줄 왼쪽 'o'; muted → 'x'.
- 좌상단 파란 루트 라벨 '|{root}'.
size='large'면 더 크게. 외부 이미지/base64 없이 순수 SVG element만 사용.
```

### Step 4 — (선택) 정적 SVG 일괄 생성 스크립트
```text
scripts/generateChordSvgs.ts 를 만들어라.
chords.ts의 ChordShape 데이터를 읽어 public/chords/{quality}/{root}.svg 파일들을 생성한다.
각 SVG는 Step 3과 동일한 스타일(가로 프렛보드, 빨강 줄/초록 프렛/검은 점/코드명)이어야 한다.
public/chords/placeholders/chord-placeholder.svg 와 public/chords/manifest.json(id/title/imageSrc)도 만들어라.
(동적 SVG로 충분하면 이 단계는 건너뛰어도 된다.)
```

### Step 5 — Header & 검색창
```text
src/components/AppHeader.tsx 를 만들어라. props: { searchTerm, onSearchChange }.
왼쪽 돋보기 아이콘(lucide-react) + 가운데 rounded-full 검색 input(연핑크 border, 연핑크 focus ring)
+ 오른쪽 'Lesson Designer'(연핑크, letter-spacing 넓게). input에 aria-label. 모바일에서 flex-wrap.
```

### Step 6 — 원형 코드 종류 버튼
```text
QualityBubble.tsx 와 QualitySelector.tsx 를 만들어라.
QualityBubble props: { quality, active, onClick }. 디자인: 완전한 원형, 흰 배경, quality별 색 border(4px),
inset+drop shadow, hover scale-105, active 시 더 큰 scale + ring, 라벨 가운데 정렬, aria-pressed.
QualitySelector: chordQualities 12개를 desktop 6×2 grid, mobile wrap으로 표시. 선택된 종류는 active.
```

### Step 7 — 코드 카드 그리드
```text
ChordCard.tsx 와 ChordGrid.tsx 를 만들어라.
ChordCard: shape를 받아 코드명 라벨(초록 점선 border 느낌) + Fretboard(size='thumb')를 표시.
  hover scale-1.04, button 또는 키보드 접근 가능.
ChordGrid: chords·selectedQuality·searchTerm을 받아 §8 필터링 결과를 표시. 없으면 EmptyState.
  grid: desktop 4열 / tablet 3열 / mobile 2열.
```

### Step 8 — 상세 화면
```text
ChordDetail.tsx 를 만들어라. props: { chord, relatedChords, onSelectChord, onBack }.
- 중앙 상단 큰 제목 박스(초록 텍스트 + 주황 border).
- 중앙에 큰 Fretboard(size='large').
- 우측에 같은 quality 다른 코드 썸네일(클릭 시 selected 변경). mobile은 하단 가로 스크롤.
- 뒤로가기 버튼 + ESC 키로 onBack.
```

### Step 9 — App.tsx 전체 연결
```text
App.tsx에서 §8 상태(searchTerm, selectedQuality, selectedChordId, uploadedChords)를 연결하라.
처음엔 QualitySelector만 강조 → 종류 클릭 시 ChordGrid → 카드 클릭 시 ChordDetail.
Header 검색은 항상 보이고, 검색어가 있으면 종류 미선택이어도 전체 검색 결과를 보인다.
Back으로 detail→grid→qualities. 컨테이너 max-w 1180px, 화면 전환에 fade/slide transition.
```

### Step 10 — 검색 동작
```text
useChordSearch 훅 또는 App 내 파생 로직으로 검색을 완성하라.
코드명(C, Am, G7), 종류명(major, minor), 태그로 필터. 대소문자 무시·부분 일치.
```

### Step 11 — IndexedDB 업로드(선택)
```text
§7.3 대로 idb 패키지로 chord image 업로드 기능을 만들어라.
파일: src/lib/chordImageStorage.ts, src/hooks/useIndexedChordImages.ts, src/components/ChordImageUploader.tsx.
DB 'guitar-chord-viewer' / store 'chord-images'. 허용 svg/png/webp, 2MB 이하, 미리보기, 저장 후 즉시 반영,
새로고침 후 재로드, 삭제 지원. Fretboard 표시 우선순위는 §7.6(IndexedDB > image > SVG).
'배포된 public 폴더는 브라우저가 직접 못 쓴다'는 설명을 코드 주석에 넣어라.
```

### Step 12 — 디자인 polish & README
```text
영상 느낌으로 다듬어라: 원형 버튼 크기/그림자/색, 카드 초록 점선 라벨, 상세 제목(초록 텍스트+주황 박스),
```

## 11. 코드 다이어그램 SVG 직접 생성용 프롬프트

정적 이미지(7.2)를 쓸 때, 이미지 자체를 AI에게 그리게 하려면:
```text
public/chords/major/c.svg 를 순수 SVG로 작성하라.
- 가로형 기타 코드 다이어그램, viewBox 520x320, 흰/투명 배경.
- 6개 줄: 연한 붉은색 가로선(위=1번줄 ~ 아래=6번줄), 오른쪽에 '1번줄'~'6번줄'.
- 프렛: 연한 초록 세로선, 하단 '1프렛/2프렛/3프렛' 초록 라벨.
- 왼쪽 너트(시작 프렛) 진하게.
- 운지점: 검은 원 + 흰 둥근 캡슐 안에 finger 번호. 개방현 'o', 뮤트 'x'.
- 좌상단 'C' 텍스트.
- 외부 이미지/base64 금지, SVG element만.
동일 스타일로 d/e/f/g/a/b.svg 도 만들고, 코드명·운지점만 해당 코드에 맞게 다르게 하라.
```

## 13. 운영 배포 주의사항
- 영상 속 UI는 **참고용**. 로고/이미지/디자인 자산은 **직접 제작**한다.
- 외부 코드 다이어그램 이미지를 그대로 복사하지 말고 **직접 SVG로 생성**하거나 라이선스가 명확한 자산만 쓴다. (동적 SVG 방식이 저작권상 가장 안전)
- 정적 배포만 하면 업로드 이미지는 다른 사용자에게 공유되지 않는다(공유하려면 서버/스토리지 필요).
- 이미지가 많아질수록 PNG보다 **SVG/WebP** 우선.
- 코드 수백 개까지는 클라이언트 필터링으로 충분, 수천 개면 인덱싱/서버 검색 고려.

## 부록 A. 표준 오픈 코드 운지표

AI가 `chords.ts`를 채울 기준 표입니다. 표기는 **저음(6번줄)→고음(1번줄)** 순서(예 `x32010`)이며,
§6.1 `positions`(객체형)로 변환할 때 **string 값을 명시**하므로 뒤집힘 걱정이 없습니다.

| 코드 | 운지 (6→1번줄) | 운지 손가락(예) |
|------|----------------|-----------------|
| C Major | `x32010` | 5번줄3(3), 4번줄2(2), 2번줄1(1) |
| D Major | `xx0232` | 3번줄2(1), 1번줄2(2), 2번줄3(3) |
| E Major | `022100` | 5번줄2(2), 4번줄2(3), 3번줄1(1) |
| F Major | `133211` (바레) | 검지 바레 1프렛, 4번줄3(3), 3번줄2(2) |
| G Major | `320003` | 6번줄3(2 또는 3), 5번줄2(1), 1번줄3(3 또는 4) |
| A Major | `x02220` | 4·3·2번줄 2프렛(1·2·3) |
| B Major | `x24442` (바레) | 5번줄 바레 2프렛, 4·3·2번줄 4프렛 |
| A minor | `x02210` | 4번줄2(2), 3번줄2(3), 2번줄1(1) |
| E minor | `022000` | 5번줄2(2), 4번줄2(3) |
| D minor | `xx0231` | 1번줄1(1), 3번줄2(2), 2번줄3(3) |
| G7 | `320001` | 6번줄3, 5번줄2, 1번줄1 |
| C7 | `x32310` | C + 3번줄3(4) |
| Dm7 | `xx0211` | 1·2번줄1(바레), 3번줄2 |
| Em7 | `022030` | E minor + 2번줄3 |

> AI에게: "위 표를 §6.1 `ChordShape.positions` 형식으로 변환해 `chords.ts`에 넣어라. (string 1=가는 줄, 6=굵은 줄 / fret 0=open / 표의 x=muted) 그리고 나머지 종류(sus4, 6, add2, m7(b5), diminish, Augment 등)도 표준 오픈 운지로 채워라."
